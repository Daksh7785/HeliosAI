## Purpose

Specifies the concrete REST API contract implementers build against — full OpenAPI schema is auto-generated by FastAPI from these definitions plus Pydantic models finalized in `05_Low_Level_Design.md`.

---

## API Conventions

- Base path: `/api/v1`.
- JSON request/response bodies, ISO-8601 UTC timestamps throughout (consistent with the UTC-only contract established in `19_Data_Synchronization.md`).
- Pagination via `limit`/`offset` query params on list endpoints.
- All endpoints require authentication (per `35_Authentication.md`) unless explicitly marked public (e.g., health check).

---

## Core Endpoints

| Endpoint | Method | Purpose |
|---|---|---|
| `/catalogue/events` | GET | List/filter nowcast events (by class, status, time range) |
| `/catalogue/events/{event_id}` | GET | Single event detail, including explainability payload |
| `/forecasts` | GET | List forecasts (by horizon, time range, min probability) |
| `/forecasts/{forecast_id}` | GET | Single forecast detail, including attribution/attention payload |
| `/lightcurve` | GET | Raw/fused light-curve series for a time range (dashboard consumption) |
| `/models` | GET | Registered model versions and current production model per engine |
| `/alerts/subscriptions` | POST/GET/DELETE | Manage webhook/email alert subscriptions |
| `/auth/token` | POST | OAuth2 password-flow token issuance |
| `/health` | GET | Public liveness/readiness check |

## Error Handling

Standard HTTP status codes; error bodies follow `{"error": {"code": "...", "message": "..."}}`. Validation errors (Pydantic) return 422 with field-level detail. Auth failures return 401/403 per `35`/`36`.

---

## Versioning

Path-based versioning (`/api/v1`, future `/api/v2` if breaking changes are needed) rather than header-based, for discoverability. Full API contract detail (request/response schemas per endpoint) is maintained in the generated OpenAPI spec once `src/` is populated.
